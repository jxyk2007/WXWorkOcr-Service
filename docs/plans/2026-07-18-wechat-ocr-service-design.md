# 微信 OCR 检测与 API 服务设计

版本号：v1.0

## 目标

构建一个基于 .NET 8 的 Windows 服务，自动探测微信 3.x 与 4.x 的安装位置和 OCR 私有组件，在组件兼容时提供图片 OCR HTTP API，在不兼容时输出可操作的诊断信息。

## 架构

解决方案由 `WeChatOcr.Core`、`WeChatOcr.Api` 和测试项目组成。Core 负责候选路径发现、组件匹配、兼容性报告及 OCR 会话抽象；Api 负责运行时配置、请求校验、串行任务队列、超时恢复和 HTTP 接口。底层通过 NuGet 包 `WeChatOcr` 调用微信 OCR，但所有依赖探测先于会话创建执行，避免以异常作为正常诊断流程。

配置使用 ASP.NET Core 配置系统的 `reloadOnChange` 能力。每个请求或新建 OCR 会话通过 `IOptionsMonitor` 获取最新配置，因此修改 `appsettings.json` 后无需重启。路径配置仅作为候选项，不覆盖自动探测；本机路径 OCR 默认关闭。

## 数据流与错误处理

诊断请求扫描配置路径、注册表卸载项和常见安装目录，查找 `mmmojo.dll`、`mmmojo_64.dll` 与用户目录中的 `WeChatOCR.exe`，生成结构化兼容性报告。OCR 请求完成格式和大小校验后进入有界串行队列，由会话服务调用微信 OCR；超时或进程断开后废弃当前会话并在下次请求时重建。

接口统一使用问题详情响应，区分组件缺失、版本不兼容、输入无效、队列已满和识别超时。微信 4.x 若仍包含兼容组件则允许实测，否则只报告不兼容，不复制或混用旧版本 DLL。

## 验证

单元测试覆盖路径去重、注册表与文件系统候选合并、兼容性判断、配置更新和输入验证。集成测试覆盖健康检查、诊断接口与 OCR 错误响应。最后运行本机 `diagnose` 命令，输出当前微信环境的真实检测结果。
